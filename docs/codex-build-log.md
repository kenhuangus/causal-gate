## 2026-07-17 — intent-assurance and software-factory gate

Request: distinguish AgentFlight from general agent observability by making intent, explicit decision context, the complete causal chain, and evidence-gated improvement the working product center. Support AI engineers and bounded recursive-improvement or software-factory workflows without claiming hidden chain-of-thought.

Decisions: added stable intent-clause identifiers; validated application-emitted `plan` and `decision` events; computed clause bindings, coverage, causal ancestry, consequential actions, and the deterministic first divergence; and added a same-fixture promotion gate with explicit checks, restored clauses, regressions, and `promote` or `hold`. Decision summaries are untrusted application records. Forbidden private-reasoning fields are rejected, and semantic model output cannot change deterministic alignment or promotion decisions.

Changes: implemented `src/agentflight/flight_record.py`, expanded typed contracts, added both fixture-scoped and administrator-scoped Intent Flight Record API access, included the record in Markdown reports, upgraded replay comparison, preserved the nine-event baseline and eight existing detector outcomes, and added dedicated backend tests. Updated the governing documents and added `docs/INTENT-ASSURANCE.md` with the product thesis, competitive boundary, and recursive-improvement safety boundary.

Independent review: a separate high-reasoning model found that the initial promotion gate did not require every baseline-divergent clause to be restored, ignored unbound candidate actions, and allowed application-asserted clause IDs to count as satisfaction. It also identified overly broad causal-chain assembly and an insufficiently strict decision-payload boundary. The gate now requires behavior-specific satisfaction for every divergent clause, zero unbound candidate actions, and an exact recorded fixture-digest replay link. Asserted bindings are observation-only, causal paths follow parent and validated evidence edges, and plan/decision payload keys are allowlisted. Regression tests cover partial restoration, unbound actions, invalid replay links, unrelated events, and cross-run evidence or clauses.

Known limits: the prototype observes only application-provided plan and decision summaries; it cannot inspect private model reasoning. The promotion gate proves the included replay and detector conditions, not general production safety. Deployed browser testing and Google Cloud deployment remain external gates in this environment.
